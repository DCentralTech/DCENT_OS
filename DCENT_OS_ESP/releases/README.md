# DCENT_axe releases

Release bundles are **generated by the signed release matrix**, not hand-placed
here. Each public Toolbox-installable target produces a signed factory `.bin`,
a signed update `.bin`, and an Ed25519 release manifest:

- `scripts/build-matrix.sh` / `scripts/build-matrix.ps1` — build the six public
  Toolbox-installable targets (Max, Ultra, Supra, Gamma, Hex Ultra, Hex Supra).
- `scripts/package-firmware.sh` / `scripts/package-firmware.ps1` — the canonical
  packaging gate: verifies the update image fits the OTA app slot and writes the
  manifest (`updateFitsSlot`, `slotSize`, SHA-256, signature). The dashboard and
  DCENT Toolbox refuse uploads when the manifest is dishonest.

A raw **ELF is never a flashable release artifact.** Flashing an ELF with a tool
that does not understand ELF (e.g. `esptool write_flash`) writes garbage and
bricks the app slot. Any `*.elf` left in this directory is a local build by-product
(gitignored via the root `*.elf` rule) — it is not a signed, slot-checked release
and must not be distributed or flashed as one.
